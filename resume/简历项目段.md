# 简历项目段

## ScentSoul - 香水数字 IP 与用户运营系统 ｜ 独立产品设计 & Vibe Coding ｜ 作品集项目

- **S：**传统香水购买体验集中在线下试香、包装审美和品牌故事上，用户购买后缺少持续互动场景，品牌难以沉淀用户偏好、情绪标签和复购线索。
- **T：**设计一套“香水 + 专属数字 IP”的用户运营系统，让用户扫码解锁与香调匹配的动态视觉、音乐和香气人格，提升购买仪式感、社交分享率和品牌会员沉淀。
- **A：**拆解“购买激活 - 香调识别 - 大模型生成 - 动画音乐呈现 - 收藏分享 - 复购运营”的完整流程，设计用户端 H5、数字 IP 卡片、运营指标看板和品牌后台指标。
- **A：**接入大模型生成链路，根据香水类型、瓶身编号、用户昵称和情绪选择，自动生成 IP 名称、人格设定、视觉参数、音乐参数、香气故事和分享文案，保证同款香水风格一致但个体细节不同。
- **A：**建立香调到视听的映射规则：柑橘水生对应水流、青草、柑橘弧线和明亮玻璃音；花香对应花瓣、薄雾和柔和 Pad；木质琥珀对应木纹、纸页、火光和低频氛围音。
- **A：**使用 Vibe Coding 完成可运行 Demo：前端用 Canvas 渲染动态香气世界，用 Web Audio 根据模型生成的 BPM、调式和亮度参数生成氛围音乐；后端用 Node.js 代理调用 OpenAI Responses API，并提供本地生成器兜底。
- **R：**完成产品方案、MVP Demo、大模型生成链路、运营玩法和简历表达；Demo 可演示“扫码激活 - 生成专属数字 IP - 动态画面与音乐呈现 - 收藏卡分享 - 运营指标沉淀”的完整闭环。

## 面试展开话术

这个项目不是简单做扫码动效，而是把香水购买后的情绪价值产品化。  
用户端获得专属数字灵魂、动态画面、音乐和收藏卡，品牌端获得扫码激活、分享率、二次唤醒、情绪偏好和香型偏好等运营数据。  
我用大模型负责生成差异化内容，用前端参数化渲染保证每一瓶都有不同结果，同时保留同款香水的统一世界观。
