# 迭代路线图

## V0.1 当前版本：可演示 MVP

已完成：

- 香水选择
- 瓶身编号
- 情绪选择
- 大模型生成入口
- 本地生成兜底
- Canvas 动态视觉
- Web Audio 香气音乐
- 收藏卡复制
- 产品闭环展示
- 运营看板和运营策略
- GitHub 开源

## V0.2：真实扫码与收藏库

计划功能：

- 生成真实二维码链接。
- 每个瓶身编号绑定唯一数字 IP。
- 增加本地收藏记录。
- 增加“我的香气灵魂图鉴”页面。

产品价值：

- 从 Demo 变成可真实使用的购买后体验。

## V0.3：品牌运营后台

计划功能：

- 香水款式配置。
- 活动主题配置。
- 稀有度规则配置。
- 扫码、分享、唤醒、复购指标看板。
- 用户情绪和香型偏好分布。

产品价值：

- 从用户端体验扩展到品牌 SaaS/运营工具。

## V0.4：AIGC 内容资产扩展

计划功能：

- 生成动态壁纸。
- 生成小红书分享文案。
- 生成短视频脚本。
- 生成节日限定卡面。
- 支持更多模型 Provider。

产品价值：

- 让数字 IP 从一次生成变成持续内容资产。

## V0.5：商业化实验

计划功能：

- 小样领取链路。
- 同系列香水推荐。
- 礼盒推荐。
- 线下快闪签到。
- 联名活动卡池。

产品价值：

- 把情绪互动转化为复购、会员和线下活动。

## 面试表达

```text
我把路线图拆成四层：先做可演示 MVP，再补真实扫码和收藏库，然后做品牌运营后台，最后做商业化和联名活动。
这样项目不是一次性页面，而是有清晰迭代路径的产品。
```
