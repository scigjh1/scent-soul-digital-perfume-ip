# 大模型生成链路

## 为什么要接大模型

如果每瓶香水都只是固定动画和固定音乐，用户会觉得它只是品牌赠品。接入大模型后，系统可以让同一款香水保持统一风格，同时为每个瓶身编号生成不同细节，形成“这是我的”感觉。

## 输入

```json
{
  "perfumeId": "nile",
  "ownerName": "郭珈豪",
  "bottleId": "SS-203915",
  "mood": "清透"
}
```

## 大模型输出

模型输出一个结构化 JSON：

- `edition`：版本名
- `rarity`：稀有度
- `ip`：电子 IP 名字、人格、口号
- `scene`：动画场景、颜色、元素、运动参数
- `music`：BPM、调式、音色、亮度、温暖度、脉冲感
- `story`：扫码开场故事、再次唤醒机制、解锁提示
- `share`：收藏卡标题、副标题、话题

## 前端如何使用

### 视觉

前端 Canvas 使用以下字段：

- `scene.palette` 控制主色。
- `scene.motifs` 控制画面元素，比如 river、grass、citrus、petal、wood、rain。
- `scene.motion.flow` 控制流动速度。
- `scene.motion.bloom` 控制绽放幅度。
- `scene.motion.sparkle` 控制亮度和闪烁。
- `scene.motion.density` 控制粒子密度。

### 音乐

Web Audio 使用以下字段：

- `music.bpm` 控制节奏。
- `music.key` 控制基础音高。
- `music.brightness` 控制滤波器亮度。
- `music.warmth` 控制 Pad 厚度。
- `music.pulse` 控制音符密度和包络。

## 安全设计

- API Key 只放在服务端环境变量，不放前端。
- 前端请求 `/api/generate`，由 `server.mjs` 代理调用 OpenAI。
- OpenAI 调用失败时，自动切换本地生成器，演示不会中断。

## 可替换模型

默认模型写在环境变量：

```powershell
$env:OPENAI_MODEL="gpt-5.5"
```

如果你想省钱或提高速度，可以换成你账号可用的轻量模型。
