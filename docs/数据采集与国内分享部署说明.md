# 数据采集与国内分享部署说明

这个 Demo 已经不是纯静态网页，而是一个带 Node 后端的动态产品原型。朋友访问页面时，系统会记录访问、生成、复制、保存和反馈提交等事件，方便你做小样本验证和简历里的转化漏斗表达。

## 会记录什么

数据写入本项目的 `data/events.jsonl`，每行是一条事件：

- `page_view`：访问页面。
- `generate_start` / `generate_success` / `generate_fallback`：开始生成、生成成功、兜底生成。
- `copy_card` / `copy_ops` / `save_png`：复制收藏卡、复制运营方案、保存动态画面。
- `feedback_submit`：提交体验评分、分享意愿、购买兴趣和一句反馈。
- `feedback_reason_select`：选择最打动用户的体验点。

默认只记录随机 `sessionId`、页面路径、浏览器信息和用户在页面里选择/填写的内容，不记录手机号、微信、邮箱等敏感信息。

## 数据接口

```text
POST /api/track
GET  /api/metrics
GET  /api/review
GET  /api/export/events.csv
GET  /api/export/events.jsonl
```

建议正式给朋友访问前设置管理口令，防止别人下载你的数据：

```powershell
$env:ADMIN_TOKEN="自己设置一个口令"
npm start
```

设置后，你自己查看数据时使用：

```text
http://localhost:4177/api/metrics?token=自己设置的口令
http://localhost:4177/api/review?token=自己设置的口令
http://localhost:4177/api/export/events.csv?token=自己设置的口令
```

你也可以直接打开产品数据后台：

```text
http://localhost:4177/dashboard.html?token=自己设置的口令
```

## 怎么发给朋友

GitHub Pages 只能放静态页面，不能把反馈写进 `data/events.jsonl`。如果你要收数据，必须运行这个 Node 服务，再把服务暴露成公网地址。

### 临时调研

适合发给 5-20 个朋友快速体验：

1. 本地运行项目：`npm start`
2. 用国内内网穿透工具把 `4177` 端口映射成公网 HTTPS 地址。
3. 把公网地址发给朋友。
4. 调研结束后导出 `/api/export/events.csv`。

可选工具：

- cpolar：https://www.cpolar.com/
- NATAPP：https://natapp.cn/

它们都属于内网穿透思路：你的电脑继续运行服务，工具提供一个临时公网地址。

### 长期作品集链接

适合写进简历或长期给面试官看：

1. 买一台国内轻量云服务器。
2. 安装 Node.js，把项目上传到服务器。
3. 配置 `LLM_PROVIDER=deepseek` 或 `qwen/kimi/zhipu`。
4. 用 `pm2` 或系统服务保持 `npm start` 常驻运行。
5. 绑定域名和 HTTPS。

参考：腾讯云轻量应用服务器 Node.js 环境文档：https://cloud.tencent.com/document/practice/1207/60266

## 简历里怎么写转化

没有真实大样本前，不要写“提升了 XX%”。可以写：

> 增加轻量埋点与反馈表单，记录访问、生成、复制、分享意愿和购买兴趣，形成“扫码激活 - 生成体验 - 收藏卡复制 - 反馈提交”的小样本验证漏斗。

收集到 10-30 个朋友样本后，再把数据换成真实口径：

- 生成完成率 = `generate_success / generate_start`
- 收藏卡复制率 = `copy_card / page_view`
- 反馈提交率 = `feedback_submit / page_view`
- 分享意愿率 = `feedback_submit` 中 `shareIntent=yes` 的占比
- 购买兴趣率 = `feedback_submit` 中 `purchaseIntent=yes` 的占比
- 核心打动点分布 = `feedback_submit` 中 `reason` 的占比

## 模型产品评审

数据后台里的“模型产品评审”会读取当前漏斗和反馈，生成：

- 产品评分
- 下一步改版优先级
- 可验证的小实验
- 面试/上线风险提醒

配置国内大模型 API Key 时，它会调用模型生成评审；没有 Key 时使用本地产品评审规则兜底，保证演示不断。
