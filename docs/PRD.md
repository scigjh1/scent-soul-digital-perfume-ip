# ScentSoul PRD

## 1. 产品定位

ScentSoul 是一套面向香水品牌的“购买后数字体验 + 用户运营”系统。用户购买香水后，通过瓶身二维码或 NFC 激活专属数字 IP，获得由大模型生成的动态视觉、音乐、香气故事和收藏卡。

## 2. 目标用户

### C 端用户

- 喜欢香水、香氛、生活方式内容的年轻用户。
- 喜欢晒图、收集卡片、分享人格测试和数字形象的用户。
- 对 ACGN、潮流、情绪消费、仪式感消费敏感的用户。

### B 端品牌

- 香水品牌、新锐香氛品牌、线下集合店。
- 想提升购买后触达、会员沉淀、社交传播和复购转化的品牌。

## 3. 核心问题

传统香水产品有三个断点：

1. 用户购买后缺少持续互动。
2. 气味难以视觉化和社交化表达。
3. 品牌难以沉淀用户偏好、情绪标签和复购线索。

## 4. 解决方案

把每一瓶香水变成可激活、可收藏、可分享、可运营的数字资产。

```text
买香水 -> 扫码激活 -> 生成数字 IP -> 动画音乐体验 -> 收藏卡分享 -> 二次唤醒 -> 复购推荐
```

## 5. MVP 功能

| 模块 | 功能 | 价值 |
| --- | --- | --- |
| 瓶身激活 | 输入/扫描唯一编号 | 建立一瓶一身份 |
| 香水选择 | 选择不同香型 | 不同香型映射不同世界观 |
| 情绪选择 | 清透、明亮、潮湿等 | 形成用户情绪标签 |
| 大模型生成 | IP、故事、视觉、音乐、运营动作 | 保证差异化和扩展性 |
| 动态视听 | Canvas 动画 + Web Audio 音乐 | 提升购买仪式感 |
| 收藏卡 | 复制/保存分享文案 | 提升社交传播 |
| 运营看板 | 激活、分享、唤醒、偏好 | 支撑运营复盘 |
| 运营策略 | 用户分层、活动主题、复购动作 | 产品功能直接连接运营动作 |

## 6. 用户故事

- 作为香水用户，我希望扫码后能看到只属于这瓶香水的数字形象，让购买更有仪式感。
- 作为内容分享用户，我希望系统帮我生成好看的收藏卡和文案，方便发小红书或朋友圈。
- 作为品牌运营，我希望看到用户偏好的香型和情绪，方便做复购推荐和会员活动。
- 作为产品经理，我希望同一套生成逻辑可以扩展到新品、节日活动和线下快闪。

## 7. 成功指标

- 激活率：扫码激活用户 / 购买用户
- 生成完成率：成功生成 IP 用户 / 扫码用户
- 分享率：保存或复制收藏卡用户 / 生成用户
- 二次唤醒率：7 天内再次进入用户 / 生成用户
- 复购意向率：点击推荐或领取小样用户 / 二次唤醒用户

## 8. 当前 Demo 范围

当前版本已经实现：

- 4 款香水世界观
- 大模型/本地生成双通道
- 动态 Canvas 场景
- Web Audio 氛围音乐
- 收藏卡复制
- 运营指标看板
- 用户分层和运营策略展示

下一步需要补：

- 国内云服务器 / Sealos / 阿里云函数计算等国内可访问部署
- 真实二维码/NFC
- 用户登录和收藏库
- 品牌后台配置能力
